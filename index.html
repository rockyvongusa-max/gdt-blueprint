01 Project Overview & Scope

The General Department of Taxation (GDT) of Cambodia is modernizing its employee management system, currently using a manual paper-based workflow, by developing a digital system called TnP (Taxation a nd Payroll) to manage employee records, salary, and performance evaluation across department heads, HR, and system administrators.


Goal: Replace paper-based workflows with a structured digital system that supports role-based access control and payroll processing.

Role-Based Access Performance Review Salary Processing GDT Internal Use
02 Functional Requirements
  • Employee Record Management: Create/Edit/Delete/View employee records with unique ID, name, DOB, department, position, hire date, status and salary information.
  • Department Management: CRUD operations for departments (Dept ID, name, description, head).
  • Position Management: CRUD for positions (Position ID, title, level, salary scale).
  • Performance Review: Annual review cycle, KPI scoring, promotion recommendations. State management: Draft → Submitted → Reviewed → Final.
  • Payroll Processing: Monthly payroll records: employee, salary, allowances, deductions, net pay. Gross = Base + Allowances − Deductions.
  • Leave Management: Leave request workflow: Pending → Approved / Rejected. Paid leave quota per employee.
  • Audit Trail: System logs all C UD operations on employee records with old/new values, actor, timestamp.
  • Search & Filter: Search employees by name, dept, position. Filter by status, department, date range.
  • Dashboard: Summary widgets: total employees, department counts, payroll totals, pending leave requests.
  • Authentication: System login with role assignment. Roles: Admin, Manager, HR, Employee.

04 Technical Stack (Proposed)
  • Frontend: React.js (SPA) or Thymeleaf (server-side rendered)
  • Backend: Java Spring Boot with MVC architecture
  • Database: PostgreSQL (primary data store) + Redis (caching)
  • Security: Spring Security with role-based access control
  • API: RESTful JSON API
  • Documentation : Swagger/OpenAPI

09 Developer Roadmap
Phase 1 · Weeks 1–3
Foundation & Database Setup
Initialize Spring Boot project > Configure PostgreSQL connection > Define JPA entities matching the data models > Run Flyway/Liquibase migrations > Verify schema generation.
Phase 2 · Weeks 4–6
Core API Layer
Implement CRUD REST controllers for Employee, Department, Position > Add Request/Response DTOs > Add input validation > Swagger documentation > Unit tests (JUnit + Mockito).
Phase 3 · Weeks 7–9
Business Logic & Payroll
Implement PayrollCalculationService > LeaveWorkflowService > PerformanceReviewWorkflowService > Calculate net_pay logic > Audit logging via AOP/observers.
Phase 4 · Weeks 10–12
Security & Auth
Configure Spring Security with JWT > Role-based endpoint protection (Admin, HR, Manager, Employee) > Stateless session management > Password encryption with BCrypt.
Phase 5 · Weeks 13–15
Frontend Integration
Connect React SPA to REST API endpoints > Implement login flow > Build employee list/detail views > Add role-based UI rendering.
Phase 6 · Weeks 16–18
Dashboard & Polish
Summary widgets (total employees, payroll totals, pending leaves) > Advanced search & filter > Pagination > Error handling > Responsive layout.
10 REST API Contracts

Base URL: /api/v1

GET    /api/v1/employees         — List all employees (paginated, filterable)
GET    /api/v1/employees/{id}      — Get employee by ID
POST   /api/v1/employees           — Create employee
PUT    /api/v1/employees/{id}       — Update employee
DELETE /api/v1/employees/{id}       — Delete employee

GET    /api/v1/departments         — List departments
POST   /api/v1/departments         — Create department
GET    /api/v1/positions          — List positions
POST   /api/v1/positions          — Create position

GET    /api/v1/payroll/{empId}/{month}  — Get payroll for employee
POST   /api/v1/payroll/generate         — Generate monthly payroll
GET    /api/v1/payroll/report           — Payroll summary report

GET    /api/v1/leave               — List leave requests
POST   /api/v1/leave               — Submit leave request
PUT    /api/v1/leave/{id}/approve — Approve leave
PUT    /api/v1/leave/{id}/reject   — Reject leave

GET    /api/v1/reviews            — List performance reviews
POST   /api/v1/reviews            — Create review
PUT    /api/v1/reviews/{id}/status — Update review status

POST   /api/v1/auth/login         — Login { email, password }
GET    /api/v1/auth/me            — Current user info

GET    /api/v1/dashboard          — Dashboard summary
