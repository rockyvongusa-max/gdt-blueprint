01 · Project Overview

What is the TnP System?

A digital employee management system to replace GDT's manual paper-based workflow — covering records, payroll, performance reviews, and leave management.

ScopeProject Goal

The General Department of Taxation (GDT) of Cambodia is modernizing its employee management system by developing a digital system called TnP (Taxation and Payroll) to manage employee records, salary, and performance evaluation across department heads, HR, and system administrators.

Role-Based Access Performance Review Leave Management Audit Trail GDT Internal
StackTechnical Stack
  • FrontendReact.js / Thymeleaf
  • BackendJava Spring Boot (MVC)
  • DatabasePostgreSQL + Redis
  • SecuritySpring Security / JWT
  • API DocsSwagger / OpenAPI
02Functional Requirements
  • Employee Record Management: CRUD with unique ID, name, DOB, department, position, hire date, status, salary.
  • Department Management: CRUD for departments (Dept ID, name, description, head).
  • Position Management: CRUD for positions (Position ID, title, level, salary scale).
  • Performance Review: Annual cycle, KPI scoring, promotion recommendations. State: Draft → Submitted → Reviewed → Final.
  • Payroll Processing: Monthly payroll: Gross = Base + Allowances − Deductions = Net Pay.
  • Leave Management: Workflow: Pending → Approved / Rejected. Paid leave quota per employee.
  • Audit Trail: Logs all CUD operations with old/new values, actor, timestamp.
  • Search & Filter: By name and department; filter by status and date range.

06 · Developer Roadmap

Implementation Phases

18-week phased rollout broken down by technical milestones.

Phase 1 · Weeks 1–3
Foundation & Database Setup
Initialize Spring Boot project → Configure PostgreSQL connection → Define JPA entities matching data models → Run Flyway/Liquibase migrations → Verify schema generation.
Phase 2 · Weeks 4–6
Core API Layer
Implement CRUD REST controllers for Employee, Department, Position → Add Request/Response DTOs → Add input validation → Swagger documentation → Unit tests (JUnit + Mockito).
Phase 3 · Weeks 7–9
Business Logic & Payroll
Implement PayrollCalculationService → LeaveWorkflowService → PerformanceReviewWorkflowService → Calculate net_pay logic → Audit logging via AOP.
Phase 4 · Weeks 10–12
Security & Auth
Configure Spring Security with JWT → Role-based endpoint protection (Admin, HR, Manager, Employee) → Stateless session → BCrypt encryption.
Phase 5 · Weeks 13–15
Frontend Integration
Connect React SPA to REST API → Implement login flow → Build employee list/detail views → Add role-based UI rendering.
Phase 6 · Weeks 16–18
Dashboard & Polish
Summary widgets (total employees, payroll totals, pending leaves) → Advanced search & filter → Pagination → Error handling → Responsive layout.
07 · API Contracts

REST API Reference

Base URL: /api/v1 — Standard response wrapper, error codes.

EndpointsREST Endpoints
GET    /api/v1/employees           — List all employees (paginated, filterable)
GET    /api/v1/employees/{id}       — Get employee by ID
POST   /api/v1/employees            — Create employee
PUT    /api/v1/employees/{id}       — Update employee
DELETE /api/v1/employees/{id}       — Delete employee

GET    /api/v1/departments           — List departments
POST   /api/v1/departments           — Create department
GET    /api/v1/positions            — List positions
POST   /api/v1/positions            — Create position

GET    /api/v1/payroll/{empId}/{month} — Get payroll for employee
POST   /api/v1/payroll/generate         — Generate monthly payroll
GET    /api/v1/payroll/report           — Payroll summary report

GET    /api/v1/leave                 — List leave requests
POST   /api/v1/leave                 — Submit leave request
PUT    /api/v1/leave/{id}/approve   — Approve leave
PUT    /api/v1/leave/{id}/reject    — Reject leave

GET    /api/v1/reviews               — List performance reviews
POST   /api/v1/reviews              — Create review
PUT    /api/v1/reviews/{id}/status   — Update review status

POST   /api/v1/auth/login            — Login { email, password }
GET    /api/v1/auth/me              — Current user info
GET    /api/v1/dashboard            — Dashboard summary
